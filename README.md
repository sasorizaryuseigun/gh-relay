# GitHub Relay

GitHub Appのインストールトークンを自動生成し、`gh`に引き継ぐラッパー。

## 概要

`gh-relay`は、GitHub CLI（`gh`）の透過的なラッパーです。事前に`GH_TOKEN`や`GITHUB_TOKEN`を用意しなくても、GitHub Appの秘密鍵からインストールアクセストークンを自動で発行し、そのトークンを使って`gh`を実行します。

認証の面倒をすべてラッパーに任せ、通常の`gh`と同じ操作性を維持します。

## 設定

設定ファイルを`~/.config/gh-relay/config.json`に配置するか、環境変数で指定します。

設定ファイルと同じディレクトリに`private-key.pem`を置くだけで動作します。

### 設定ファイル

```json
{
  "appId": 123456,
  "installId": 654321
}
```

| フィールド       | 必須   | 説明                                                         |
| ---------------- | ------ | ------------------------------------------------------------ |
| `appId`          | はい   | GitHub AppのID                                               |
| `installId`      | はい   | インストールID                                               |
| `privateKeyPath` | いいえ | 秘密鍵ファイルのパス（相対パスは設定ファイルのディレクトリ基準） |

秘密鍵の内容を直接設定ファイルに書くことはできません。環境変数`GH_PRIVATE_KEY`を使ってください。

### 秘密鍵の解決

秘密鍵は次の優先順位で解決されます。

1. 環境変数`GH_PRIVATE_KEY`（最優先）
2. 環境変数`GH_PRIVATE_KEY_PATH`（相対パスは設定ファイルのディレクトリ基準）
3. 設定ファイルの`privateKeyPath`（相対パスは設定ファイルのディレクトリ基準）
4. デフォルト: `{設定ディレクトリ}/private-key.pem`

最もシンプルな構成は、設定ファイルと同じディレクトリに`private-key.pem`を置くことです。

### 環境変数

| 変数名                      | 説明                                                               | 優先順位                         |
| --------------------------- | ------------------------------------------------------------------ | -------------------------------- |
| `GH_APP_CONFIG`             | 設定ファイルのパス（デフォルト: `~/.config/gh-relay/config.json`） | 最高                             |
| `GH_APP_ID`                 | GitHub App ID                                                      | 環境変数 > 設定ファイル          |
| `GH_INSTALL_ID`             | インストールID                                                     | 環境変数 > 設定ファイル          |
| `GH_PRIVATE_KEY`            | 秘密鍵の内容（文字列）                                             | 環境変数のみ（設定ファイル不可） |
| `GH_PRIVATE_KEY_PATH`       | 秘密鍵ファイルのパス（相対パスは設定ファイルのディレクトリ基準）   | 環境変数 > 設定ファイル          |
| `GH_TOKEN` / `GITHUB_TOKEN` | 手動でトークンを指定する場合（設定不要）                           | 最高                             |

`GH_TOKEN`または`GITHUB_TOKEN`がすでに設定されている場合、トークンの自動生成は行われず、そのまま`gh`に渡されます。

## インストール

### ビルド

```bash
bun install
bun run build
```

`./dist/gh`に実行バイナリが出力されます。PATHの通った場所に配置してください。

```bash
cp ./dist/gh ~/.local/bin/gh
```

このラッパーは`gh`と同じ名前でインストールすることを想定しています。既存の`gh`より先にPATHで見つかるようにしてください。

## 使い方

通常の`gh`とまったく同じように使えます。

```bash
gh pr list
gh issue view 42
gh repo create my-new-repo --public
```

`GH_TOKEN`や`GITHUB_TOKEN`が設定されていない場合、呼び出すたびに設定からトークンを生成して`gh`に渡します。

ラッパーは実行時に`PATH`から自身を取り除き、本物の`gh`を呼び出します。本物の`gh`の終了コードはそのまま継承されます。
